
# Skills Required — CuraFlow Frontend

Claude should apply the following skills when working on this project.

This is a professional SaaS interface.

## 1. SaaS Dashboard Design

Understand modern SaaS UI patterns:
- sidebar navigation
- dashboard metrics
- card layouts
- operational tables
- analytics panels
- activity feeds

Good reference products:
- Stripe
- Linear
- Notion
- Vercel

Key principles:
- strong hierarchy
- minimal clutter
- consistent spacing

## 2. Design System Thinking

A good design system includes:
- typography
- color palette
- spacing scale
- border radius
- shadow system
- reusable components

Spacing should follow an 8px rhythm.

Example spacing scale:
4px, 8px, 16px, 24px, 32px, 48px, 64px

## 3. Professional Card-Based Layouts

Cards should include:
- title
- content
- optional actions

Cards use:
- subtle shadows
- rounded corners
- consistent padding

## 4. Data Table Design

Tables must support:
- search
- filtering
- sorting
- row actions

Avoid overcrowded tables.

## 5. Visual Hierarchy

Example structure:

Page Title  
Page Description

Section  
  Card  
  Card

Section  
  Table

Use typography and spacing to guide the eye.

## 6. Typography Best Practices

Use modern SaaS fonts:

Inter  
system-ui  
Segoe UI  
Roboto

Hierarchy:
H1: page title  
H2: section title  
H3: card title

## 7. Calm Health-Tech Aesthetic

The UI must feel:
- calm
- trustworthy
- professional

Avoid neon colors or playful UI styles.

## 8. Responsive Layouts

Layouts must adapt across devices.

Patterns:
- sidebar collapses on mobile
- cards stack vertically
- tables scroll horizontally

## 9. Accessibility Awareness

Follow accessibility basics:
- adequate color contrast
- semantic HTML
- readable font sizes
- labeled inputs

## 10. Component Reuse

Reusable components should be used whenever possible.

Examples:
StatCard  
AlertCard  
ServiceCard  
ProgressWidget  
TableToolbar

## 11. Operational UX

Interfaces must support:
- speed
- clarity
- monitoring
- decision making

Avoid decorative UI that slows users down.

## 12. Product Thinking

Before generating UI ask:
- What decision is the user making?
- What information is essential?
- What actions are possible?
